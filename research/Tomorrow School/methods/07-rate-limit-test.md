# Method 07: Rate Limit Test

**Status:** ✅ Проверено  
**Date:** 2026-07-14  

## Цель
Проверить наличие rate limiting на validation API для оценки возможности брутфорса.

## Процесс

### Sequential: 10 requests
```
10 requests in 932ms = ~10 req/sec
```

### Parallel: 20 concurrent requests
```
20 requests in 322ms = ~62 req/sec
```

### Parallel: 100 concurrent requests
```
100 requests in 1357ms = ~73 req/sec
```

## Результат
Rate limiting **отсутствует**. Можно отправлять ~70 запросов/сек параллельно.

## Оценка брутфорса

### Полный перебор (36^5 = 60,466,176)
```
60.5M / 70 req/sec = 864,374 сек = ~10 дней
```

### Только буквы (26^5 = 11,881,376)
```
11.9M / 70 req/sec = 169,734 сек = ~47 часов
```

### Оптимизированный (только частые паттерны)
```
~1M / 70 req/sec = ~4 часа
```

## Вывод
Брутфорс теоретически возможен, но требует длительного времени. Оптимальная стратегия: начать с буквенных кодов (26^5), затем буквы+цифры (36^5).
